# Детектив — гра про злочин

Гра-прототип: гравець отримує історію вбивства з підказками (приладдя, місце, докази), обирає підозрюваних, допитує їх і має одну спробу назвати вбивцю.

## Стек

- **Next.js 15** (App Router)
- **Supabase** — зберігання справ, підозрюваних, свідчень
- **TypeScript**, **Tailwind CSS**

## Як запустити

1. Клонуйте репозиторій і встановіть залежності:

```bash
cd detective-game
npm install
```

2. Створіть проєкт у [Supabase](https://supabase.com), скопіюйте `.env.example` в `.env.local` і вкажіть:

```
NEXT_PUBLIC_SUPABASE_URL=https://ваш-проєкт.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш-anon-key
```

3. У Supabase SQL Editor виконайте міграцію та seed (у такому порядку):

- `supabase/migrations/001_initial.sql` — створення таблиць
- `supabase/seed.sql` — тестові справи та підозрювані

4. Запустіть dev-сервер:

```bash
npm run dev
```

Відкрийте [http://localhost:3000](http://localhost:3000).

## Структура даних (Supabase)

- **suspects** — підозрювані (ім’я, прізвище, вік, робота, хобі, погана звичка, біографія, фото).
- **cases** — справи: вступ, місце знаходження тіла, знаряддя злочину, докази, складність (3–5 підозрюваних), `killer_id`, мотив, текст зізнання.
- **case_suspects** — зв’язок справи з підозрюваними: хто в справі, чи вбивця, текст свідчення (підказка або «вода»).

Сценарій обирається випадково; для кожної справи набір підозрюваних і логіка (хто вбивця, свідчення) задані в БД, тому історія залишається унікальною та логічною.

## Логіка гри

1. **Головна** — кнопка «Почати нову справу» випадково обирає справу з Supabase і перенаправляє на `/game?caseId=...`.
2. **Сторінка справи** — показує текст історії (вступ + місце + знаряддя + докази) і картки підозрюваних (ім’я, робота, хобі, погана звичка тощо).
3. **Картка підозрюваного** — по кліку відкривається особиста справа: біографія, фото, кнопка «Допросити». Після натискання показується свідчення по цій справі (або підказка на вбивцю, або «вода» — залежить від запису в `case_suspects`).
4. **Звинувачення** — кнопка «Звинуватити» відкриває вибір підозрюваного. Одна спроба: якщо обрано вбивцю — показується зізнання та мотив; якщо ні — показується прикольне повідомлення про провал і можливість почати знову.

## Додавання контенту

- **Нові справи:** додайте рядки в `cases` (включно з `killer_id`) і відповідні рядки в `case_suspects` для кожного підозрюваного цієї справи (з `testimony_text` та `is_killer`).
- **Нові підозрювані:** додайте запис в `suspects`; фото можна зберігати в Supabase Storage і вказувати `image_url`.
- Тексти з оригінального `index.js` (перші/другі/треті частини історій, знаряддя, докази) можна використати як шаблони для нових записів у `cases` або розширити seed.

## Примітки

- Зображення підозрюваних: у seed використані шляхи на кшталт `/suspects/oliver.jpg`. Їх потрібно завантажити в Supabase Storage (наприклад, bucket `suspects`) і підставити повні URL у `image_url`, або замінити на placeholder.
- Якщо Supabase не налаштований, на головній після «Почати нову справу» з’явиться повідомлення про відсутність даних і підказка запустити seed.
